### How we work
We co-author RFCs in CoMLRL, prototype quickly in notebooks, and move promising ideas into the gallery as soon as they run end to end. Contributors rotate between agent training, evaluation science, and system orchestration so every release reflects real coordination pressure.

If you would like to collaborate, open an issue on GitHub or reach out at `hello@openmlrl.org` with a short note about the agent behaviors or toolchains you are interested in stress testing.
